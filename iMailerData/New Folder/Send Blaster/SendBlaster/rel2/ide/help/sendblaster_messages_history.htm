History

This window allows you to see the list of all sent messages. In case a mailing operation was not completed (for example if your SMTP server won't allow sending more than a certain number of e-mails in a short period of time) it is possible to resume sending from the point where it was interrupted, sending the message to all the recipients not included in the original sending operation.

Each mailing operation gets marked with the send date, the list name to which the message was sent and the subject.

Click on the E-mail preview button (or select the corresponding command with a right mouse click) to show the message sent for the selected entry.

The Address list allows you to see a list of all addresses to which the message was sent. All addresses to which the message was successfully sent are shown in green, while addresses shown in red are the ones to which delivery failed.

The Log button shows the send log, which is the dialog that took place with the SMTP server (or with the destination servers in case of a direct send).

In case one or more addresses were not included in the mailing operation, the last column of the list (for incomplete entries) will show the button . Clicking it will enable you to complete the sending operation using the current sending options, as specified in then Send tab. The message will then be sent to all recipients that did not yet receive it (the red addresses in the Address list window).

History items can be deleted by selecting them (use the Ctrl key on the keyboard to select multiple elements or use Shift to select a group) and by clicking on the Delete items button.

Delivered and undelivered addresses can be exported as a text file using Export delivered and Export undelivered buttons. Exporting undelivered addresses may also be useful when you want to purge lists from bad addresses: you can later import the file selecting remove or unsubscribe options.

     

When is it useful to use the History functionalities?

Completing a message sending operation through the History functionalities is a great way to work around SMTP server restrictions that wouldn't otherwise allow sending several messages in a short time span. SMTP server restrictions could be such that, despite settings in Advance settings of the Send tab, won't allow completion of a sending operation in a single step. For instance, in a situation where you have a server that won't allow sending more than 50 messages per hour and a distribution list with 100 addresses, only the first half will go through during the first attempt. After a one-hour wait the operation can be completed from the History window.

Exporting undelivered (red) addresses may also be useful when you want to purge lists from bad addresses.
