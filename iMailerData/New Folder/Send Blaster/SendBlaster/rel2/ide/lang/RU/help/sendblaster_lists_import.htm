Импортирование

Вы можете импортировать данные из внешнего источника в текущий список. Четыремя возможными источниками являются:

  • Вы можете импортировать форматированный текстовый файл (.csv, .txt), находящийся на локальной машине или в локальной сети, выбрав Импортировать из внешнего текстового файла. «Форматированный текст» — формат обмена, широко используемый табличными процессорами и базами данных: большинство программ, которые попадают в эту категорию, могут экспортировать данные в этом формате.
     
  • Вы можете импортировать данные с веб-сайта, выбравИмпортировать данные с веб-сайта через http://. Процесс выбора очень похож на импорт форматированного текста, с тем отличием, что файл находится не на локальном диске или локальной сети, а на сервере, доступном по протоколу HTTP. Данные могут быть импортированы из скрипта, генерирующего их динамически (например, на ASP или PHP). Этот метод позволит Вам импортировать содержимое находящейся в Интернете базы данных, у которой есть экспортирующий скрипт. Веб-ссылка будет указана в качестве источника данных.
     
  • Вы можете импортировать данные из буфера обмена Windows (после помещения их туда другой программой), выбрав Импорт из буфера обмена.
     
  • Вы можете импортировать данные из Адресной книги Outlook Express (Адресной книги Windows),выбрав Импортировать из Outlook Express.

Данные из внешних источников могут быть использованы для одной из следующих операций:

  • Они могут быть добавлены в выбранный список, если указать Добавить в список и подписать адреса.
     
  • Они могут быть удалены из выбранного списка при указании опции Удалить новые адреса из списка. Все адреса, указанные в источнике данных, будут удалены.
     
  • Можно прекратить подписку на список рассылки (без удаления их из базы), выбрав опцию Прекратить подписку новых адресов. Эта операция похожа на описанную выше, но адреса не удаляются из списка, а просто помечаются как отписанные от него. Этот метод позволит Вам управлять данными списка, избегая добавления адресов в почтовые операции.

После выбора списка, источника данных и действия, щёлкните по кнопке Импорт. Появится окно, позволяющее Вам указать несколько опций, необходимых для завершения операции импортирования:

  • Путь к файлу импорта или Адрес файла импорта в случае импортирование из Интернета (импортирование из буфера обмена или Outlook Express не требует указания пути).
     
  • Формат данных (не требуется для импортирования из Outlook Express): если Вы выберите Разделённый, Вы также должны будете указать Разделитель полей, и Первая строка содержит имена полей. Если выбрать Свободный формат, то будут импортированы только email-адреса (распознанные автоматически), и спецификация данных не потребуется.
     
  • Если Вы совершаете типичную операцию импортирования (то есть, добавление данных или Добавить в список и подписать новые адреса), будут доступны две дополнительных опции для индикации действия, которое будет производиться, если новый адрес уже присутствует в списке назначения: если выбрать Не добавлять адреса при дублировании, уже имеющиеся в списке адреса будут игнорироваться (в противном случае, они будут добавляться, порождая дубликаты); если выбрать Подписывать адреса, уже существующие в списке, если адрес уже есть в списке, но не подписан (см. Управление списками), адрес не будет импортирован, но существующий такой же адрес будет помечен как подписанный.
     
  • В следующем шаге Вы должны будете указать соответствие полей между полями во внешнем источнике данных и полями SendBlaster. the Next button, the importing

Слепки особенно полезны при импортировании: если Вы успешно импортировали файл, сделайте слепок Вашей конфигурации, чтобы сохранить использованные настройки; Вы сможете использовать их ещё раз в дальнейшем парой щелчков мышью.

Осторожно: корректное указание поля внешнего источника, содержащего email-адрес, особенно важно, потому что:

  1. все внешние записи, у которых в поле для почтового адреса нет корректного распознаваемого почтового адреса, будут проигнорированы.. Следовательно, если указать неправильное поле, ни одна запись не будет содержать корректный почтовый адрес, и все внешние данные будут игнорироваться.
  2. Операция сравнения между внешними данными и данными списка нужна, чтобы проверить, что адрес уже был включён в список, проводится только между полями с email-адресами.
