Управление возвращёнными письмами

Возвращённое письмо (bounce-back) — письмо, не достигшее адресата и отправленное отправителю обратно почтовым сервером с указанием ошибки. Обычно это происходит, когда Вы производите рассылку через SMTP-сервера: если адрес не существует, SMTP-сервер (у которого нет способа узнать это) всё равно примет сообщение, но вернёт письмо обратно потом.

Две основные группы возвращаемых писем:

  • Жёсткие возвращения: email не может быть доставлено из-за перманентной ошибки (например, несуществующий адрес)
  • Мягкие возвращения: временное состояние, мешающее доставке письма (например, почтовый сервер реципиента временно занят)

SendBlaster может автоматически распознавать основную массу* возвращённых писем и отписывать или удалять адреса из выбранного списка.

Возвращённые письма отправляются на адрес почтового отправителя, который может и не быть адресом, используемым для подписки и прекращения подписки (см. Управление подписками). Следовательно, необходимо сообщать программе, который адрес следует проверять.

     

Который адрес лучше ввести?

Помните, что возвращённые письма приходят на адрес отправителя, а не на адрес, используемый для подписки. Допустим, Вы выбрали «subscriptions@mysite.com» в качестве Вашего адреса управления подпиской для Списка 1, и потом начали отправлять сообщения с адреса «newsletter@mysite.com». Возвращённые письма будут доставлены на «newsletter@mysite.com», и этот адрес Вы должны будете ввести в секцию Управление возвращёнными письмами для Списка 1.

      

Управляющий адрес каждого списка устанавливается путём ввода корректных параметров подключения к аккаунту (учётной записи) POP3, как в обычной почтовой программе: кроме email-адреса надо указать получающий сервер (POP3-сервер), номер Порта (если номер порта не будет указан, он будет установлен в 110), Имя пользователя и Пароль. Поле для галочки SSL указывает на включение SSL-соединения, необходимого некоторым серверам для работы (типичным примером является Gmail). В поле Таймаут можно указать число секунд неактивности, после которого соединение будет закрыто. Для проверки параметров щёлкните по кнопке Проверка соединения (при этом сообщения не будут приниматься).

Подсказка: Кнопка Профили позволяет управлять несколькими POP3-настройками. Информация об аккаунтах может быть сохранена для нескольких профилей, и все настройки POP3 могут быть сохранены и загружены за пару щелчков мышью. Эта функция особенно полезна при рассылке писем по чьему-то поручению, или просто чтобы быстро сохранить профиль, который Вы примените потом в том же или другом списке.

>Отмечая поле Оставлять копию сообщений на сервере после получения, Вы указываете SendBlaster, чтобы он скачивал сообщения без удаления их с сервера. Сообщения будут доступны для последующего использования. Новые сообщения будут игнорироваться.

При отмеченном поле Удалять только новые сообщения о возврате SendBlaster будет удалять только такие сообщения, оставляя всё прочее на серверем.

Если указан Удалять адресата возвращённого письма, адреса, письма с которых возвращаются, будут удалены из выбранного списка. Если нет, они будут просто отписаны от списка.

Отмечая Обрабатывать жёсткие возвраты и Обрабатывать мягкие возвраты Вы можете выбирать, какие виды возвратов обрабатывать. Если не указать ничего, то никакие возвраты не будут обрабатываться.

Если нажать на кнопку Получить, новые сообщения будут скачаны, и выбранный список рассылки (а также прочие списки, использующие этот адрес для получения возвратов) будет обновлён.

Осторожно: если скачивание новых сообщений кажется слишком медленным, то это может значить, что на сервере скопилось очень много сообщений: отключите опцию Сохранять копию сообщений на сервере и щёлкните по Получить (или примите письма своим обычным почтовым клиентом). После скачивния сообщения будут удалены с сервера, и последующие сеансы приёма писем будут проходить быстрее.

 

*Новые форматы возвращённых писем могут быть указаны в экране Дополнительные настройки.
