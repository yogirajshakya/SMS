Διαχείριση bounce-backs

Ένα bounced ηλεκτρονικό μήνυμα (ή bounce-back) είναι αυτό το οποίο δεν φτάνει ποτέ στα εισερχόμενα μηνύματα του παραλήπτη και στέλνεται πίσω, κάνει bounce back στον αποστολέα με ένα μήνυμα σφάλματος που εξηγεί στον αποστολέα οτι το ηλεκτρονικό μήνυμα δεν έφτασε ποτέ επιτυχώς στον προορισμό του. Τυπικά, αυτό συμβαίνει όταν στέλνουμε μηνύματα χρησιμοποιώντας διακομιστές Smtp: αν μια διεύθυνση δεν υπάρχει, ο διακομιστής Smtp (ο οποίος δεν έχει κανένα τρόπο να το γνωρίζει) θα αποδεχτέι το μήνυμα έτσι κι αλλιώς, αλλά θα το κάνει bounce αργότερα.

Υπάρχουν δύο βασικοί τύποι bounce-backs:

  • Hard bounces: το ηλεκτρονικό μήνυμα δεν μπόρεσε να παραδωθεί λόγο κάποιου μόνιμου σφάλματος (π.χ. μη υπαρκτή διεύθυνση)
  • Soft bounces: Μια προσωρινή κατάσταση υπάρχει η οποία κάνει την παράδοση του μηνύματος αποτυχημένη (π.χ. ο διακομιστής του παραλήπτη είναι προσωρινά απασχολημένος)

Το SendBlaster μπορεί αυτόματα να αναγνωρίσει τα περισσότερα bounced ηλεκτρονικά μηνύματα και να ακυρώσει ή να διαγράψει τις επιλεγμένες απο την λίστα bounced διευθύνσεις.

Τα Bounced ηλεκτρονικά μηνύματα θα αποστέλλονται στην διεύθυνση του αποστολέα, η οποία μπορεί να μην είναι η ίδια διεύθυνση που χρησιμοποιήθηκε για τις εγγραφές συνδρομής ή για τις ακυρώσεις (δείτε Διαχείριση εγγραφών). Άρα είναι απαραίτητο να πείτε στο πρόγραμμα ποιά διεύθυνση θα πρέπει να ελεγχθεί.

     

Ποιά διεύθυνση θα πρέπει να εισάγω;

Θυμηθείτε οτι τα bounce ηλεκτρονικά μηνύματα στέλνονται στην διεύθυνση του αποστολέα, όχι στην διεύθυνση που έχει χρησιμοποιηθεί για τις εγγραφές. Ας υποθέσουμε οτι επιλέξατε "subscriptions@mysite.com" ως την διεύθυνση διαχείρισης εγγραφών για την Λίστα 1, και οτι αργότερα στείλατε τα μηνύματά σας απο την διεύθυνση "newsletter@mysite.com" . Τα Bounce-back μηνύματα θα παραδωθούν στην "newsletter@mysite.com", και αυτή είναι η διεύθυνση την οποία θα πρέπει να ρυθμίσετε στον τομέα Διαχείριση των bounce-backs για την Λίστα 1.

      

Η διεύθυνση που πρέπει να ελεγχθεί, ρυθμίζετε πληκτρολογώντας τις σωστές παραμέτρους για να υπάρξει πρόσβαση στον POP3 λογαριασμό της, ακριβώς όπως σε κάθε λογισμικό πελατών ηλεκτρονικού ταχυδρομείου: εκτός απο την διεύθυνση (E-mail) θα πρέπει να διευκρινίσετε ποιός είναι ο διακομιστής παραλαβής (POP3 Διακομιστής ), και θα πρέπει αν χρειαστεί να συμπεριλάβετε τον αριθμό θύρας "διακομιστής:μορφή" θύρας, το Όνομα χρήστη και τον Κωδικό. Το SSL κουτάκι ενεργοποιεί την SSL επικοινωνία, η οποία μπορεί να χρειάζεται απο κάποιους pop3 διακομιστές (ένα τυπικό παράδειγμα είναι ο διακομιστής του Gmail). Στο Timeout πεδίο μπορείτε να πληκτρολογήσετε τον αριθμό των δευτερολέπτων αδράνειας μετά απο τον οποίο η σύνδεση θα τερματιστεί. Για να επαληθεύσετε τις παραμέτρους απλά πατήστε στο κουμπί Test connection (κάνενα μήνυμα δεν θα ληφθεί).

Όταν επιλέξετε Κράτηση αντίγραφου των μηνυμάτων στον διακομιστή, το SendBlaster θα λαμβάνει τα νέα μηνύματα χωρίς να τα διαγράφει απο τον διακομιστή. Τα μηνύματα τότε θα είναι διαθέσιμα για μελλοντική χρήση. Τα καινούρια μηνύματα θα αγνοούνται.

Αν επιλεχθεί η Διαγραφή bounced διευθύνσεων , τότε οι bounced διευθύνσεις θα διαγράφονται απο την επιλεγμένη λίστα. Αν όχι, τότε θα ακυρώνονται απλώς απο αυτήν.

Επιλέγοντας Επεξεργασία των hard bounces και Επεξεργασία των soft bounces μπορείτε να διαλέξετε ποιούς τύπους bounce (δείτε παραπάνω) θέλετε να επεξεργάζονται, φυσικά αν καμία εναλλακτική δεν επιλεχθεί δεν θα υπάρξει επεξεργασία των bounced ηλεκτρονικών μηνυμάτων.

Πατώντας στο κουμπί Λήψη , τα καινούρια μηνύματα θα λαμβάνονται και η επιλεγμένη λίστα διανομής θα ενημερώνεται (όπως και όλες οι άλλες λίστες που χρησιμοποιούν την ίδια ηλεκτρονική διεύθυνση για bounce-backs).

Προειδοποίηση: αν η λήψη των καινούριων μηνυμάτων είναι αργή, αυτό μπορεί να σημαίνει οτι έχουν στοιβαχθεί πολλά μηνύματα στον διακομιστή του ηλεκτρονικού ταχυδρομείου : απενεργοποιήστε την επιλογή Κράτηση αντίγραφου των μηνυμάτων στον διακομιστή και πατήστε στην Λήψη (ή εκετελέστε την λήψη στο σταθερό σας ηλεκτρονικό ταχυδρομείο). Τα μηνύματα, μόλις ληφθούν, θα διαγραφούν απο τον διακομιστή και οι επόμενες λήψεις θα είναι γρηγορότερες.
