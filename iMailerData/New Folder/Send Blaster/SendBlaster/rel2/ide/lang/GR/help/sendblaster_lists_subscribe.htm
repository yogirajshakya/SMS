Διαχείριση εγγραφών

Το SendBlaster μπορεί αυτόματα να διαχειριστεί τις εγγραφές (και τις διαγραφές) των λιστών διανομής. Η κάθε λίστα μπορεί να είναι συνδεδεμένη με μια ηλεκτρονική διεύθυνση: όταν ένα μήνυμα με τη λέξη "Εγγραφή" στην γραμμή θέματος στέλνετε σε αυτη την ηλεκτρονική διεύθυνση ο αποστολέας προστίθεται στην λίστα. Αν το θέμα είναι "Διαγραφή" αντ'αυτού, ο αποστολέας διαγράφεται (αν ήταν εγγεγραμμένος). Το SendBlaster λαμβάνει νέα μηνύματα απο τον POP3 λογαριασμό, επαληθεύει το θέμα και ενημερώνει την σχετική λίστα διανομής.

Προειδοποίηση: Η επιλεγμένη διεύθυνση για την διαχείριση εγγραφών θα πρέπει να επιτρέπει στα POP3 μηνύματα την λήψη (το ίδιο πρωτόκολλο που χρησιμοποιείται και απο τους σταθερούς πελάτες ηλεκτρονικών διευθύνσεων). Δεν είναι δυνατό να χρησιμοποιούνται διευθύνσεις που μπορούν να διαχειριστούν μόνο μέσω του δικτύου (για παράδειγμα Hotmail υπηρεσίες).

Η κάθε λίστα θα ανατεθεί σε μια συγκεκριμένη ηλεκτρονική διεύθυνση. Παρ΄όλα αυτά, είναι δυνατό να ανατεθεί η ίδια ηλεκτρονική διεύθυνση σε περισσότερες απο μια λίστες. Σε αυτή την περίπτωση, όταν μια λίστα ενημερώνεται, όλες οι άλλες λίστες που χρησιμοποιούν την ίδια διεύθυνση Εγγραφής/Διαγραφής θα ενημερώνονται και αυτές. Για παράδειγμα, αν έχετε δημιουργήσεις 2 λίστες και τις έχετε ονομάσει "Λίστα 1" και "Λίστα 2" και είναι και οι δύο συνδεδεμένες με την ίδια ηλεκτρονική διεύθυνση διαχείρισης, λαμβάνοντας τα μηνύματα για να ενημερώσετε την "Λίστα 1", "η Λίστα 2" θα ενημεωθεί επίσης.

Η διεύθυνση διαχείρισης κάθε λίστας καθορίζεται πληκτρολογώντας τις σωστές παραμέτρους για να γίνει πρόσβαση στον POP3 λογαριασμό, όπως ακριβώς στο λογισμικό ενός πελάτη ηλεκτρονικής διεύθυνσης: εκτός απο την διεύθυνση (E-mail) πρέπει να προσδιορίσετε και τον διακομιστή λήψης(POP3 Διακομιστή), ο οποίος, αν χρειαστεί, θα συμπεριλαμβάνει και τον αριθμό θύρας στο στήσιμο "διακομιστής:θύρα" , το Όνομα χρήστη και τον Κωδικό. Το κουτάκι SSL δίνει την δυνατότητα για SSL επικοινωνία, η οποία μπορεί να χρειάζεται απο κάποιούς pop3 διακομιστές (ένα τυπικό παράδειγμα είναι ο διακομιστής του Gmail). Στο πεδίο Timeout μπορείτε να πληκτρολογήσετε τον αριθμό των δευτερολέπτων αδράνειας πριν κλείσει η σύνδεση. Για να επαληθεύσετε τις παραμέτρους απλά πατήστε στο κουμπί Δοκιμή σύνδεσης (κανένα μήνυμα δεν θα ληφθεί).

Επιλέγοντας το Κράτηση αντίγραφου των μηνυμάτων στον διακομιστή, το SendBlaster θα λάβει τα νέα μηνύματα χωρίς να τα διαγράψει απο τον διακομιστή. Τα μηνύματα τότε θα είναι διαθέσιμα για μελλοντική χρήση.Τα καινούρια μηνύματα θα αγνοούνται.

Η Ενημέρωση όλων των πεδίων είναι μια πολύ ισχυρή επιλογή: όταν επιλέγεται, όχι μόνο το θέμα, αλλά όλο το κεντρικό μήνυμα θα επεξεργάζεται, αν αυτό περιέχει όνομα πεδίου: σειρές αξιών, και θα ενημερώνονται αναλόγως, όχι μόνο τα αρχεία επαφών ηλεκτρονικών διευθύνσεων αλλά και άλλα πεδία (δείτε τον παρακάτω πίνακα).

Το μειονέκτημα είναι οτι η λήψη και η επεξεργασία των μηνυμάτων θα είναι σημαντικά πιο αργή, και για αυτό το λόγο σας προτείνουμε το Ενημέρωση όλων των πεδίων να επιλέγεται όταν χρειάζεται πραγματικά, δηλαδή όταν η ιστοσελίδα σας παράγει ηλεκτρονικά μηνύματα εγγραφών που δεν περιέχουν μόνο διευθύνσεις αλλά και άλλα πεδία επίσης.

Πατώντας στο κουμπί Λήψη , τα νέα μηνύματα θα ληφθούν και οι επιλεγμένες λίστες διανομής θα ενημερωθούν (καθώς επίσης και όλες οι άλλες λίστες που χρησιμοποιούν την ίδια ηλεκτρονική διεύθυνση διαχείρισης).

     

Ποιά διεύθυνση θα πρέπει να χρησιμοποιήσω;

Σας προτείνουμε να εισάγετε μια διαφορετική ηλεκτρονική διεύθυνση εγγραφής/διαγραφής για κάθε λίστα διανομής, που σημαίνει, μια διεύθυνση που δεν χρησιμοποιείται σταθερά για να λαμβάνεται και να στέλνετε ηλεκτρονικά μηνύματα. Αυτό θα μειώσει το ρίσκο του να λαμβάνετε spam (το οποίο θα αγνοείται) και θα είναι πιο εύκολο να γίνει η διαχείριση των λίστων. Σε ένα τέτοιο περιβάλλον, δεν θα είναι απαραίτητο να ενεργοποιήσετε την επιλογή Κράτηση αντίγραφου των μηνυμάτων στον διακομιστή αφού τα ηλεκτρονικά μηνύματα δεν θα πρέπει να διαβάζονται απο άλλα λογισμικά. Με αυτό τον τρόπο, η λήψη μηνυμάτων θα είναι γρηγορότερη, και πάνω απ΄όλα, ο χώρος του λογαριασμού θα διατηρηθεί. Ορισμένες υπηρεσίες έχουν ένα όριο στο συνολικό όγκο χώρου που μπορεί να καταλάβει ένα μήνυμα, και αν χώρος που επιτρέπεται γεμίσει, δεν παραδίδονται άλλα μηνύματα.

Είναι επίσης πιθανό, αλλά δεν σας το συμβουλεύουμε, να χρησιμοποιήσετε μια "σταθερή" διεύθυνση, η οποία να χρησιμοποιείται και απο κάποιον άλλο πελάτη ηλεκτρονικού ταχυδρομείου για να στέλνετε και να λαμβάνετε μηνύματα. Με αυτόν τον τρόπο, είναι απαραίτητο να ενεργοποιήσετε την επιλογή Κράτηση αντίγραφου των μηνυμάτων στον διακομιστή (αλλιώς, μόλις ληφθούν τα μηνύματα διαγράφονται απο τον διακομιστή). Και κάτι ακόμη πιο σημαντικό είναι οτι ακόμη και ο κανονικός πελάτης ηλεκτρονικού ταχυδρομείου ρυθμίζετε ανάλογα, για να επιτρέπει στο SendBlaster να διαβάζει τα μηνύματα (στο Outlook Express αφού διαλέξετε ένα λογαριασμό επιλέξετε Ιδιότητες / Εξελιγμένες ρυθμίσεις / Κράτηση αντίγραφου των μηνυμάτων στον διακομιστή).

      

Προειδοποίηση: αν η λήψη των νέων μηνυμάτων είναι κάπως αργή, αυτό μπορεί να υποδηλώνει οτι έχουν στοιβαχτεί πολλά μηνύματα στον διακομιστή ηλεκτρονικών μηνυμάτων: απενεργοποιήστε την επιλογή Κράτηση αντίγραφου των μηνυμάτων στον διακομιστή πατήστε στην Λήψη (ή πραγματοποιήστε την λήψη στον σταθερό σας πελάτη ηλεκτρονικού ταχυδρομείου). Τα μηνύματα, μόλις ληφθούν, θα διαγραφούν απο τον διακομιστή και οι επόμενες λήψης θα είναι πιο γρήγορες.

     

Πώς δουλεύει η "Η Ενημέρωση όλων των πεδίων" ;

Ενώ "η κανονική " επεξεργασία στο SendBlaster λαμβάνει και επεξεργάζεται μόνο τα θέματα των μηνυμάτων, η επιλογή Ενημέρωση όλων των πεδίων θα κάνει το SendBlaster να λάβει και να επεξεργαστεί ολόκληρα τα μηνύματα, θα ψάξει για γραμμές στο όνομα πεδίου: σχήματα αξιών μέσα στο κυρίως μήνυμα, και θα ενημερώσει ανάλογα τα πεδία.

Ας υποθέσουμε οτι έχετε δύο ονόματα πεδίων "Επώνυμο" και "Πόλη", αν ένα μηνυμα ληφθεί απο το "aaa@aaa.com" με θέμα "Εγγραφή" και το ακόλουθο μήνυμα ως κυρίως (παρακαλώ σημειώστε οτι η κάθε γραμμή περιέχει διαφορετικό πεδίο):

Επίθετο: Σμίθ
Πόλη: Νέα Υόρκη

μια νέα επαφή "aaa@aaa.com" προστίθεται στην λίστα, που έχει το "Σμιθ" ως "Επώνυμο" και την "Νέα Υόρκη" ως "Πόλη".

Κάποια άλλα συντακτικά σχήματα, που χρησιμοποιούν τις ίδιες Ετικέτες με την σύνθεση μηνύματος, επιτρέπονται επίσης: #ετικέτα#: αξία και #αριθμός τομέα#: αξία όπως στα ακόλουθα παραδείγματα:

#Επώνυμο#: Σμίθ
#πόλη#: Νέα Υόρκη

και

#2#: Σμιθ
#3# : Νέα Υόρκη
