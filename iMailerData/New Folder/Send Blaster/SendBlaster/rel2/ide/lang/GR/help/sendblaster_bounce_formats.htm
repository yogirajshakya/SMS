Συνήθεις μορφές Bounce-backs

Οι Bounce-back μορφές ηλέκτρονικού ταχυδρομείου έχουν αρχικά σχεδιαστεί για να διαβάζονται απο ανθρώπους, και όχι απο μηχανές, και έτσι δεν υπάρχει μια τυποποιημένη μορφή ή κανόνας. Το SendBlaster μπορεί να επεξεργαστεί τις πιο κοινές μορφές bounce-back χρησιμοποιώντας την εσωτερική βάση δεδομένων του, αλλά μπορεί να αντιμετωπίσετε κάποιες μορφές που δεν αναγνωρίζονται από το λογισμικό. Αν συμβεί αυτό, μπορείτε να προσθέσετε μια καινούρια χειροκίνητα, συνήθη μορφή στην bounce-backs βάση δεδομένων του SendBlaster.

Μια συνήθης μορφή αποτελείται απο κάποιους κανόνες, και πρέπει όλοι να πληρούνται για να θεωρηθεί ένα μήνυμα ως bounce-back (με άλλα λόγια, ο λογικός φορέας των κανόνων είναι AND).

Όταν θέλετε να προσθέσετε μια Καινούρια μορφή, πρώτα απ'όλα θα πρέπει να προσδιορίσετε κάποιο σχέδιο στο Θέμα ή σε άλλες κεφαλίδες, οι οποίες θα ανγνωρίζουν το μήνυμα μοναδικά ως bounce-back (ως Hard bounce, αν επιλέξετε το κουτάκι, ή ως soft αν δεν το επιλέξετε). H αναπτυσσόμενη λίστα των κεφαλίδων εξυπηρετεί μόνο ως παράδειγμα, μπορείτε χειροκίνητα να προσθέσετε μια συγκεκριμένη κεφαλίδα πληκτρολογώντας το όνομά της στο πεδίο Header και το επιθυμητό σχήμα στο πεδίο Value .

Ένα σχήμα συνήθως περιλαμβάνει ορισμένα σύμβολα υποκατάστασης (το σύμβολο "*" που σημαίνει "οποιονδήποτε χαρακτήρα"): για παράδειγμα, αν γράψετε "*γνωστοποίηση*παράδοσης*" σαν σχήμα Θέματος, οποιοδήποτε μήνυμα θα περιέχει τις λέξεις "παράδοση" και "γνωστοποίηση" στο θέμα του - π.χ. "γνωστοποίηση αποτυχίας παράδοσης στο xxx@yyy.com" - θα θεωρηθεί ως bounce-back, υπό τον όρο ότι επίσης ταιριάζει με τους άλλους κανόνες που έχετε καθορίσει. Αν χρησιμοποιείτε "γνωστοποίηση παράδοσης", χωρίς σύμβολα υποκατάστασης, μόνο τα μηνύματα που έχουν ακριβώς τις λέξεις "γνωστοποίηση παράδοσης" σαν θέμα τους, θα επεξεργάζονται ως bounce-backs - "αποτυχία παράδοσης στο xxx@yyy.com" δεν θα θεωρηθεί ως έγκυρο θέμα.

Μπορείτε επίσης να εισάγετε κάποιες συγκεκριμένες λέξεις (που να χωρίζονται με κόμμα) που θα πρέπει να βρίσκονται στο κεντρικό μήνυμα: αν δεν βρεθεί καμία απο αυτές τις λέξεις, το μήνυμα δεν θα θεωρείτε ως bounce-back. Για παράδειγμα, τα περισσότερα bounce-backs περιέχουν την λέξη "αιτία" (που ακολουθείται απο ένα επεξηγηματικό κείμενο για τον λόγο που δεν παραδόθηκε το μήνυμα). Σημειώστε οτι η λίστα των λέξεων δεν είναι σχήμα (δεν μπορείτε να χρησιμοποιήσετε σύμβολα υποκατάστασης) και ο λογικός φορέας των κανόνων είναι OR.

Αν ένα συγκεκριμένο μήνυμα πληροί όλους τους κανόνες που έχετε ορίσει, θα επεξεργαστεί ως bounce-back, και όλες οι διευθύνσεις ηλεκτρονικού ταχυδρομείου που θα βρεθούν μέσα στο μήνυμα θα διαγραφούν, ακόμη κι αν βρεθούν στη λίστα για την οποία λαμβάνετε τα bounces. Μπορεί να θέλετε να αποκλείσετε τις διευθύνσεις του τυπικού διαχειριστή συστήματος χρησιμοποιώντας κάποιο σχήμα.

     

Παράδειγμα

Ας πούμε οτι θέλετε τα μηνύματα να επεξεργάζονται ως bounce-backs όταν:

α) έχουν τις λέξεις "ανεπίδοτο" και "μήνυμα" στο θέμα τους,
β) είναι απο μια "postmaster@..." διεύθυνση,
γ) ο τύπος περιεχομένου της κεφαλίδας τους είναι "γνωστοποίηση παράδοσης" ή "γνωστοποίηση αποτυχίας παράδοσης"
δ) περιέχουν την λέξη "αιτία" ή "σφάλμα" στο κεντρικό μήνυμα

και προφανώς εσείς δεν θέλετε να διαγραφεί η διεύθυνση του αποστολέα ("postmaster@...") , καθώς η πραγματική διεύθυνση βρίσκεται κάπου αλλού μέσα στο μήνυμα.

Εδώ μπορείτε να δείτε πως θα είναι η κάθε μορφή:

Hard bounce :   (ελεγμένο)
Σχήμα του θέματος:   *ανεπίδοτο*μήνυμα*
1. Κεφαλίδα:   Απο
  Αξία:   *διαχειριστής συστήματος*
2. Κεφαλίδα:   Τύπος περιεχομένου
  Αξία:   *γνωστοποίηση*παράδοσης*
Το κεντρικό μήνυμα περιέχει :   αιτία, σφάλμα
Αποκλεισμός :   postmaster@*

Και εδώ είναι μια πολύ πιο απλή μορφή, που θα επεξεργάζεται όλα τα μηνύματα που περιέχουν την λέξη "ανεπίδοτο" στο θέμα τους (και θα αποκλείει τις διευθύνσεις postmaster@... and majordomo@... ):

Hard bounce :   (ελεγμένο)
Σχήμα του θέματος:   *ανεπίδοτο*
1. Κεφαλίδα:   (κενό)
  Αξία:   (Κενό)
2. Κεφαλίδα:   (κενό)
  Αξία:   (κενό)
Το κεντρικό μήνυμα περιέχει :   (κενό)
Αποκλεισμός :   postmaster@*, majordomo@*
