Συχνές ερωτήσεις

Όταν στέλνω μήνυμα σε πολλούς αποδέκτες, μπορούν να δουν όλοι τις διευθύνσεις όλων;

Όχι, το SendBlaster στέλνει το κάθε μήνυμα ξεχωριστά: η ιδιωτικότητα του αποδέκτη διατηρείται. Το να χρησιμοποιείτε το SendBlaster για να διαχειριστείτε πολλαπλές αποστολές, είναι λοιπόν χρήσιμο για να αποφύγετε την κατά λάθος αποκάλυψη των ηλεκτρονικών διευθύνσεων άλλων ανθρώπων, κάτι το οποίο θα αποτελούσε παραβίαση των κανόνων προστασίας των προσωπικών δεδομένων.
 

Πώς μπορώ να φιλτράρω μια λίστα;

Χρησιμοποιώντας τις λειτουργίες διαλογής / ομαδοποίησης στον τομέα Διαχείριση λιστών , και το να ενεργοποιήσετε / απενεργοποιήσετε προσωρινά μια ομάδα διευθύνσεων, θα σας επιτρέψει να στείλετε ένα μηνύματα σε μια υποομάδα των διευθύνσεων της τρέχουσας λίστας.

Πώς μπορώ να εισάγω στο κεντρικό μήνυμα έναν σύνδεσμο διαγραφής απο την λίστα διευθύνσεων;

Το SendBlaster μπορεί να διαχειριστεί τις λίστες διανομής του λαμβάνοντας μηνύματα απο μια αποκλειστική διεύθυνση ηλεκτρονικού ταχυδρομείου, διαφορετική για κάθε λίστα (δείτε Διαχείριση εγγραφών). Ως εκ τούτου, με τη χρήση ετικετών, είναι απλό να εισαγάγετε ένα σύνδεσμο στον κωδικό του HTML :

<a href="mailto:#managingemail#?Subject=Unsubscribe">Unsubscribe</a>

Ο κωδικός του HTML θα πρέπει να εισαχθεί στον τομέα HTML του μηνύματος κάνοντας κλικ στο Εμφάνιση κωδικού, αλλιώς μπορεί να εισαχθεί σαν σύνδεσμος, χρησιμοποιώντας το κουμπί Σύνδεσμος στην μπάρα διαμόρφωσης μηνύματος (σε αυτή την περίπτωση το κείμενο mailto:#managingemail#?Θέμα=Διαγραφή θα πρέπει να εισαχθεί στο παράθυρο του συνδέσμου, μόλις ο σύνδεσμος αυτός δημιουργηθεί, και το κείμενο που εμφανίζεται μπορεί να τροποποιηθεί σε κάτι σαν “Διαγραφή” ή σε ένα παρόμοιο κείμενο).

Όταν στέλνω ένα μήνυμα στην λίστα διευθύνσεων, κάποια απο τα μηνύματα δεν παραδίδονται και επιστρέφουν με σφάλμα.

Όταν κάνουμε αποστολή σε πολλές διευθύνσεις είναι λογικό το να μην παραδίδονται όλα τα μηνύματα: κάποιες απο τις διευθύνσεις μπορεί να έχουν απενεργοποιηθεί ή να έχουν μπλοκάρει προσωρινά, οι διακομιστές των ηλεκτρονικών ταχυδρομείων μπορεί να είναι υπερφορτωμένοι, η σύνδεση μπορεί να έχει διακοπεί στιγμιαία ή να είναι αργή.

Αν ένα μεγάλο ποσοστό των απεσταλμένων μηνυμάτων είναι αργό, ή αν κανένα μήνυμα δεν στέλνεται, είναι απαραίτητο να ελέγξετε τις Ρυθμίσεις Αποστολής.

Για την ακρίβεια, ίσως είναι απαραίτητο να αλλάξετε τον τρόπο αποστολής (απευθείας ή μέσω του SMTP) και, αν ο SMTP διακομιστής σας χρησιμοποιεί φίλτρα antispam, μειώστε τον αριθμό συνδέσεων και αυξήστε τα κενά ανάμεσα στα μηνύματα. Ο SMTP διακομιστής μπορεί να έχει περιορισμούς που θα σας εμποδίσουν να στείλετε περισσότερα απο έναν συγκεκριμένο αριθμό μηνυμάτων μέσα σε συγκεκριμένο χρόνο (για παράδειγμα κάποιος διακομιστής μπορεί να μην επιτρέπει να σταλούν πάνω απο 50 ηλεκτρονικά μηνύματα την ώρα), σε αυτή την περίπτωση, μετά την πρώτη αποστολή, θα ολοκληρώσετε την λειτουργία αργότερα χρησιμοποιώντας τις Λειτουργίες Ιστορίας.

Αν οι περιορισμοί του antispam του SMTP διακομιστή δεν μπορούν να παραμεριστούν και οι λειτουργίες αποστολής γίνονται μόνο με πολύ αργές ρυθμίσεις, μπορείτε να δοκιμάσετε την άμεση αποστολή ή να χρησιμοποιήσετε ένα “πιο ανεκτικό” SMTP διακομιστή (στο Διαδίκτυο είναι πιθανό να γίνεται συνδρομητής, ακόμη και με πολύ χαμηλή τιμή, σε πολύ αποτελεσματικές SMTP υπηρεσίες).

Αν το πρόβλημα εμφανιστεί όταν χρησιμοποιείτε την άμεση αποστολή, μπορεί να είναι απαραίτητο να προσδιορίσετε με το χέρι την IP διεύθυνση του DNS διακομιστή.
 

Έχω χάσει τον κωδικό και δεν έχω πρόσβαση στο πρόγραμμα. Τι μπορώ να κάνω;

Θα πρέπει να ξέρετε οτι δεν είναι δυνατή η ανάκτηση των χαμένων δεδομένων. Η ακόλουθη διαδικασία θα σας δώσει απλά την δυνατότητα να έχετε πάλι πρόσβαση στο λογισμικό.

1. Απεγκαταστήστε το λογισμικό απο το Control Panel των Windows.

2. Απομακρύνετε με το χέρι τον φάκελο εγκατάστασης (π.χ. C.\Program files\SendBlaster), συμπεριλαμβανομένων και των υποφακέλων. Για λόγους ασφαλείας, όταν το λογισμικό απεγκαθίσταται, η βάση δεδομένων δεν απομακρύνεται αυτόματα (και για αυτό τον λόγο θα πρέπει να την διαγράψετε με το χέρι).

3. Εγκαταστήστε πάλι το λογισμικό.
