Histórico

Esta janela permite ver a lista de todas as mensagens enviadas. No caso de uma operação de mailing não ter terminado (por exemplo, caso o seu servidor de SMTP não permita enviar mais o que um determinado número de e-mails num curto intervalo de tempo) é possível retomar o envio a partir do momento em que foi interrompido, enviando a mensagem para todos os destinatários para quem a mensagem não tinha sido enviada anteriormente.

Cada operação de mailing é marcada com a data de envio, o nome da lista para a qual a mensagem foi enviada e o assunto.

Clique no botão Pré-visualizar e-mail (ou seleccione o comando correspondente através de um clique com o botão direito do rato) para exibir a mensagem enviada para a entrada seleccionada.

A Lista de endereços permite ver uma lista de todos os endereços para os quais a mensagem foi enviada. Todos os endereços para os quais a mensagem foi enviada com sucesso serão aqui exibidos a verde, caso contrário serão exibidos a vermelho.

O botão Registo exibe o registo de envio, que é o diálogo que houve entre o servido SMTP (ou com os servidores de destino, caso seja um envio directo).

No caso de um ou mais endereços não serem incluídos na operação de envio, a última coluna da lista (para entradas incompletas) exibirá o botão . Ao clicar nele activará a retoma da operação de envio utilizando as opções de envio actuais, tal como indicado na aba Envio. A mensagem será então enviada para todos os destinatários que ainda a não tenham recebido (os endereços a vermelho na janela da lista de endereços).

Os itens do histórico podem ser apagados, bastando que para isso os seleccione (através da tecla Ctrl do teclado para seleccionar vários elementos ou através da tecla Shift para escolher um grupo) e clicando no botão Apagar itens.

Os endereços entregues e não entregues podem ser exportados como ficheiro de texto através dos botões Exportar entregues e Exportar não entregues. Exportar os endereços não entregues pode também ser útil para quando quiser limpar os endereços incorrectos das listas: mais tarde poderá importar o ficheiro seleccionando as opções exportar remover ou cancelar subscrição.

     

Quando é útil utilizar as funcionalidades do histórico?

Terminar uma operação de envio através das funcionalidades do histórico é uma óptima forma de contornar as limitações dos servidores SMTP que não permitem enviam várias mensagens num curto período de tempo. As restrições do servidor SMTP podem ser tais que, apesar das Configurações avançadas da aba Enviar, não permitam que a operação de envio termine num só passo. Por exemplo, numa situação em que exista um servidor que não permita enviar mais do que 50 mensagens por hora e uma lista de distribuição com 100 endereços, apenas a primeira metade irá ser enviada durante a primeira tentativa. Depois de uma hora de espera, a operação poderá ser retomada a partir da janela do histórico.

A exportação de endereços não entregues (vermelhos) também pode ser útil para limpar os endereços incorrectos das listas.
