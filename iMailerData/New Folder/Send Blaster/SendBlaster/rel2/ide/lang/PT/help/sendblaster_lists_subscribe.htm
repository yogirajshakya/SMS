Gerir subscrições

O SendBlaster pode gerir subscrições (e cancelamentos) das listas de distribuição. Cada lista pode ser associada a um endereço de e-mail: quando é enviada para esse endereço de e-mail uma mensagem com "Subscrever" na linha do assunto, o remetente é adicionado à lista. Se o assunto for "Cancelamento", o remetente é removido (caso antes se encontrasse subscrito). O SendBlaster descarrega as novas mensagens da conta POP3, verifica o assunto e actualiza a respectiva lista de distribuição.

Nota: O assunto padrão ("Subescrever" para subscrições, e "Cancelar Subscrição" para cancelamentos) podem ser mudadas na janela Configurações Avançadas.

Aviso: O endereço escolhido para gerir as subscrições deverá permitir o descarregar de mensagens através de POP3 (o mesmo protocolo utilizado pelos clientes de e-mail comuns). Não é possível utilizar endereços que apenas podem ser geridos através da web (por exemplo, um e-mail do Hotmail).

Normalmente a cada lista será atribuído um endereço de e-mail dedicado. Contudo, é possível atribuir o mesmo endereço de e-mail a mais do que uma lista. Neste caso, quando uma lista for actualizada, todas as outras listas que utilizarem o mesmo endereço de subscrição/cancelamento serão actualizadas. Por exemplo, se você criar 2 listas chamadas "Lista 1"e "Lista 2" e ambas forem geridas através do mesmo endereço de e-mail, ao descarregar as mensagens para actualizar a "Lista 1", a"Lista 2" também seria actualizada.

O endereço de gestão de cada lista é definido digitando os parâmetros correctos para aceder à conta POP3, tal como num software cliente de e-mail: para além de especificar o endereço (E-mail) terá também de especificar o servidor de entrada (servidor POP3), o número de porta (se nenhuma for especificada, a porta utilizada será a com o número 110), o Utilizador e a Senha. A caixa de verificação SSL activa a comunicação SSL, que pode ser necessária para alguns servidores pop3 (um exemplo típico disso é o servidor do Gmail). No campo Tempo de expiração você pode digitar o número de segundos e inactividade após os quais a conexão será terminada. Para verificar os parâmetros, clique no botão Testar conexão (nenhuma mensagem será descarregada).

Sob determinadas circunstâncias, díficeis de prever, o seu servidor POP3 server poderá precisar de abrandar o ritmo de descarregamento de mensagens; se estiver com dificuldades, pode tentar activar a opção Slower (não há um caminho recorrente aqui, tente os dois métodos e ver qual deles funciona melhor).

Dica: O botão Perfis permite gerir diferentes contas POP3. A informação de cada conta pode ser guardada para vários perfis. Portanto, todas as configurações POP3 são guardadas/carregadas com um par de cliques. Esta funcionalidade é especialmente útil quando envia mensagens em nome de alguém; ou apenas para rapidamente guardar um perfil para usar mais tarde na mesma lista, ou com outra.

Se assinalar a opção Manter cópia das mensagens no servidor, o SendBlaster irá descarregar as novas mensagens sem as apagar do servidor. As mensagens ficarão disponíveis para usos futuros. As novas mensagens serão ignoradas.

Se assinalar a opção Apenas eliminar novas mensagens de subscrição e cancelamento, o SendBlaster irá eliminar apenas as mensagens reconhecidas como subscrições ou cancelamento; esta opção é realmente útil se utiliza o mesmo endereço de email para receber emails que não são de gestão, tais como respostas de leitores e outras mensagens.

A opção Actualizar todos os campos é extremamente poderosa: quando assinalada, não só o assunto mas também o corpo da mensagem será processado; se o corpo da mensagem contiver linhas de nomedecampo: valor, não só o endereço de e-mail, mas também os outros campos serão actualizados (ver caixa abaixo).

A desvantagem é que o descarregar das mensagens e o seu processamento ficará significativamente mais lento; por isso, sugerimos que a opção Actualizar todos os campos apenas seja assinalada quando for realmente necessária, isto é, quando o ser website gerar e-mails de subscrição que contenham não só endereços, mas também outros campos.

Ao clicar no botão Descarregar, as novas mensagens serão descarregadas e a lista de distribuição seleccionada será actualizada (assim como as outras listas que utilizem o mesmo endereço de e-mail de gestão).

     

Que endereço deverei usar?

Aconselhamos que dedique um diferente endereço de e-mail de subscrição/cancelamento para cada lista de distribuição, isto é, um endereço que não seja usado para outros fins. Isto irá minimizar o risco de receber spam (que será ignorado) e tornará a gestão das listas mais fácil. Num ambiente deste género não seria necessário activar a opção Manter cópia das mensagens no servidor, uma vez que os e-mails não seriam lidos por outros softwares. Desta forma, o descarregar das mensagens será mais rápido e o espaço da conta será preservado. Alguns serviços possuem um limite para o espaço que as mensagens podem ocupar e se a quota for ultrapassada, não serão entregues mais mensagens.

Também é possível, mas não aconselhável, utilizar um endereço "standard", também utilizado por outro cliente de e-mail para enviar e receber mensagens. Desta forma, é absolutamente necessário activar a opção Manter cópia das mensagens no servidor ou a opção Apenas eliminar novas mensagens de subscrição e cancelamento (caso contrário, assim que as mensagens forem descarregadas elas serão eliminadas do servidor). E ainda mais importante é que se o software de cliente de e-mail não for configurado correctamente, de modo a permitir que o SendBlaster leia as mensagens (no Outlook Express, depois de seleccionar uma conta, escolha Propriedades / Configurações avançadas / Manter cópia das mensagens no servidor).

      

Aviso: se o descarregar das novas mensagens parecer lento, pode ser sinal que o servidor de e-mail possui muitas mensagens em fila de espera: desactive a opção Manter cópia das mensagens no servidor e clique em Descarregar (ou descarregue-as no seu cliente de e-mail). As mensagens, assim que descarregadas, serão removidas do servidor e os futuros descarregamentos serão mais rápidos.

     

Como funciona a opção "Actualizar todos os campos"

O procedimento "normal" faz com que o SendBlaster apenas descarregue e processe os assuntos das mensagens, enquanto que a opção Actualizar todos os campos irá permitir que o SendBlaster descarregue e processe os corpos das mensagens, procure linhas no formato nomedecampo: valor e actualize os campos correspondentes.

Suponha que possui dois campos chamados "Apelido" e "Cidade"; se uma mensagem for recebida de "aaa@aaa.com" com "Subscrever" no assunto e o seguinte corpo de mensagem (note que cada linha contém um campo diferente):

Apelido: Correia
Cidade: Porto

será adicionado à lista o novo contacto "aaa@aaa.com" com o Apelido "Correia" e "Porto" como "Cidade".

Também são permitidos outros formatos de sintaxe, utilizando as mesmas Tags tal como na escrita de mensagens: #tag#: valor e #fieldnumber#: valor tal como nos seguintes exemplo:

#apelido#: Correia
#cidade#: Porto

e

#2#: Correia
#3# : Porto
