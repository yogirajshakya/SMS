Gérer les rebonds

Un courriel renvoyé (ou un rebond) est un message qui n’a jamais été livré à la boite destinataire et qui est renvoyé à l’expéditeur avec un message d’erreur qui averti l’expéditeur que le courriel n’a jamais été transmit correctement. Normalement, ceci arrive quand vous envoyez votre mailing en utilisant des serveurs SMTP : si une adresse n’existe pas, le serveur SMTP (qui ne peut pas le savoir) acceptera le message, mais le renvera plus tard.

Il y a deux types principaux de rebonds:

  • Rebonds durs: le courriel ne peut pas être livré à cause d’une erreur permanente (ex. une adresse inexistante)
  • Rebonds souples: une condition temporaire fait échoué la transmission (ex. le serveur destinataire peut être saturé)

SendBlaster peut reconnaitre automatiquement la plupart* des courriels renvoyés et désinscrire et effacer les adresses renvoyées dans la liste sélectionnée.

Les courriels renvoyés seront envoyés à l'adresse de l’expéditeur, qui ne peut pas être la même adresse utilisée pour les inscriptions et les désinscriptions (voir Gérer les inscriptions). Donc il est nécessaire de dire au logiciel quelle adresse doit être utilisée.

     

Quelle adresse dois-je mettre?

Rappelez-vous que les courriels renvoyés sont envoyés à l’adresse de l’expéditeur, non à l’adresse utilisée pour les inscriptions. Supposez que vous choisissez "inscriptions@monsite.com" comme adresse de gestion des inscriptions pour la Liste 1, puis envoyez vos mailings depuis l’adresse "newsletter@monsite.com". Les rebonds seront donc livrés à "newsletter@monsite.com", et c’est l’adresse que vous devez configurer dans la section Gérer les bounce-backs pour la Liste 1.

      

L’adresse à contrôler est saisie en mettant les paramètres adéquats pour accéder à son compte POP3, comme pour tout logiciel client de courriel: en plus de l’adresse (Courriel) vous devez spécifier le serveur destinataire (Serveur POP3), le numéro du Port (si aucun port n'est spécifié, le port par défaut 110 sera utilisé), le NomUsager et le MotDePasse. La case à cocher SSL active la communication SSL, qui peut être requise par quelques serveurs POP3 (les serveur Gmail sont un exemple typique). Dans le champ TempsMort vous pouvez insérer le nombre de secondes d’inactivité après lesquelles la connexion sera terminée. Pour vérifier les paramètres vous n’avez qu’à cliquer sur le bouton Test de Connexion (aucun message ne sera téléchargé).

Notez bien: le bouton Profiles permet de gérer plusieurs configurations POP3. L'information du compte de multiples profiles peut être enregistrée, donc tous les paramètres POP3 sont sauvegardés/téléchargés à l'aide de quelques clics. Cette particularité est très utile lorsque vous envoyez des courriels de la part d'autrui.

Lorsque Maintenir une copie des messages sur le serveur est coché, SendBlaster téléchargera tous les nouveaux messages sans les effacer du serveur. Les messages seront disponibles pour un usage futur. Les nouveaux messages seront ignorés.

En cochant Effacer seulement les nouveaux messages de rebonds SendBlaster effacera seulement ces messages, laissant les autres messages sur le serveur.

Si Effacer les adresses renvoyées est coché, les adresses renvoyées seront effacées de la liste sélectionnée. Sinon, ils seront juste désinscrites de la liste.

En cochant Traiter les rebonds durs ou Traiter les rebonds souples , vous pouvez choisir quels types de rebonds (voir ci-dessous) doivent être traités; évidemment si aucune option n'est choisie, les messages renvoyés ne seront pas traités.

En cliquant sur le bouton Télécharger, les nouveaux messages seront téléchargés et la liste de distribution sélectionnée sera mise à jour (comme toutes les autres listes qui utilisent la même adresse pour les messages de renvois).

Notez Bien: si le téléchargement de nouveaux messages paraît être trop lent, ceci peut indiquer que sur le serveur de courriel, les messages se sont empilés : désactivez l’option Tenir une copie des messages sur le serveur et cliquez sur Télécharger (ou exécutez le téléchargement sur votre client standard de messagerie). Les messages, une fois téléchargés, seront effacés du serveur et les téléchargements suivants seront plus rapides.
