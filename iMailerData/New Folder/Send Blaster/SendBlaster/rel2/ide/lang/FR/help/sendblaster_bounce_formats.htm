Formats personalisés des Rebonds

Les formats de courriel de rebonds ont été initialement conçus pour être lus par des humains, et non par des machines, il n'y a donc pas de format standard ou de règle. SendBlaster peut traiter la plupart des formats de rebonds communs en utilisans sa base de données interne, mais vous pourriez rencontrer certains formats qui ne sont pas reconnus par le logiciel. Si cela se produit, vous pouvez ajouter manuellement un nouveau format personnalisé à la base de donnée des rebonds de SendBlaster.

Un format personnalisé se compose de quelques règles ; chacun d'entre elles doivent être remplies pour qu'un message soit considérée comme un rebond (en d'autres termes, l'opérateur logique des règles est AND).

Lorsque vous ajoutez un Nouveau format, avant toute chose, vous devez identifier un certain modèle dans le Sujet ou dans les autres en-têtes, qui identifie exclusivement le message comme étant un rebond (un Rebond dur, si vous cochez la case, ou un rebond doux si vous la laissez décochée). La liste déroulante des en-têtes ne sert que d'exemple, vous pouvez manuellement ajouter un en-tête spécifique en tapant le nom de l'en-tête dans le champ En-tête et le modèle désiré dans le champ Valeur.

Un Modèle inclue typiquement quelques jokers (le signe "*" signifie "quelconque caractère"): par exemple, si vous entrez "*livraison*notification*" dans le modèle du Sujet , tout message contenant les mots "livraison" et "notification" dans son sujet - i.e. "livraison à xxx@yyy.com échec de la notification" - sera considéré tel un rebond, à condition que cela réponds égalment aux autres règles que vous pourriez avoir définies. Si vous utilisez "notification de livraison", sans de jokers, seuls les messages ayant exactement "notification de livraison" dans leur sujet seront traités tels des rebonds - "livraison à xxx@yyy.com échec de la notification" ne sera pas considéré comme un sujet valide.

Vous pouvez également entrer des mots spécifiques (séparés par des virgules), qui doivent être trouvés dans le corps du message: si aucun de ces mots n'est trouvé, le message n'est pas considéré comme un rebond. Par exemple, la plupart des rebonds contient le mot «raison» (suivi d'un texte qui explique pourquoi le message n'a pas été livré). S'il vous plaît noter que la liste de mots n'est pas un modèle (vous ne pouvez pas utiliser de jokers) et que l'opérateur logique de la liste est OR.

Si un message donné répond à toutes les règles que vous avez défini, il est traité comme un rebond, et toutes les adresses courriels trouvées à l'intérieur du message seront désabonnées si elles se trouvent aussi dans la liste des rebonds que vous téléchargez pour utilisation. Vous pourriez vouloir exclure les adresses typiques de postmaster en utilisant un modèle.

     

Un exemple

Disons que vous voulez les messages à être traités comme des rebonds s'ils:

a) ont les mots "non distribuable" et "message" dans leur sujet;
b) et viennent d'une adresse "postmaster@...";
c) leur en-tête Type-de-Contenu est "notification de la livraison" ou bien "échec de la notification de la livraison"
d) contiennent le mot "raison" ou "erreur" dans le corps principal du message

et vous ne voulez pas évidemment que l'adresse de l'expéditeur ("postmaster@...") soit désabonnée, car la véritable adresse inacessible est ailleurs dans le corps du message.

Voici à quoi pourrait ressembler le format:

Rebond dur :   (coché)
Modèle du sujet:   *non distribuable*message*
1. En-tête:   De
  Valeur:   *postmaster*
2. En-tête:   Contenu-Type
  Valeur:   *livraison*notification*
Corps du message contient :   raison, erreur
Sauf :   postmaster@*

Et voici un format beaucoup plus simple, qui va traiter tous les messages ayant les mots "non distribuable" dans le sujet (et qui va exclure aussi bien les adresses postmaster@... que majordomo@... ):

Rebond dur :   (coché)
Modèle du sujet:   *non distribuable*
1. En-tête:   (vide)
  Valeur:   (vide)
2. En-tête:   (vide)
  Valeur:   (vide)
Corps du message contient :   (vide)
Sauf :   postmaster@*, majordomo@*
