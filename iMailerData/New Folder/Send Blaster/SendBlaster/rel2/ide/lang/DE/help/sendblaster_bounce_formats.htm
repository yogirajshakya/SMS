Nutzerspezifische Formate für unzustellbare E-Mails

Formate für unzustellbare Emails wurden ursprünglich entworfen, um von Programmen und nicht um von Menschen gelesen zu werden. Aus diesem Grund gibt es keine standardisierten Formate oder Regeln. SendBlaster ist aufgrund der integrierten Datenbank in der Lage die meisten Formate für unzustellbare Emails zu bearbeiten. Sollten Sie trotzdem auf Formate stoßen, welche von der Software nicht erkannt werden, können Sie manuell ein neues nutzerspezifisches Format zu SendBlaster hinzufügen.

Ein kundenspezifisches Format basiert auf den vom Anwender festgelegten Regeln; diese müssen in der Reihenfolge den Auswahlkriterien entsprechen, welche die Nachricht als Rückläufer einstufen. (Mit anderen Worten: der logische Operator der Regeln ist UND)

Wenn Sie ein neues Format hinzufügen möchten, müssen Sie zuerst einige Muster für Themen oder Titel festlegen, welche eine Nachricht unverwechselbar als Unzustellbar definieren (als HARTE Unzustellbarkeit, falls sie das Kontrollkästchen auswählen, oder als WEICHE Unzustellbarkeit, falls es ungeprüft bleiben soll). Die Dropdown-Liste der Titel dient nur als Beispiel. Sie können spezifische Titel manuell hinzufügen, indem Sie den Namen des Titels in das HEADER-Feld eintragen und das gewünschte Muster im INHALT Feld angeben.

Ein Muster enthält normalerweise einige Platzhalter (Ein "*" Symbol steht für "alle Zeichen") wie zum Beispiel: Wenn Sie "*delivery*notification*" als Betreffsmuster eingeben, wird jede Nachricht die das Wort "delivery" und "notification" im Betreff enthält - wie beispielsweise "delivery to xxx@yyy.com notification failure" - als ein bounce-back betrachtet.
Weitere vom Anwender erfasste Regeln werden ebenfalls unterstützt und bei der Analyse der Nachrichten nach bounce-backs mit einbezogen. Falls Sie "delivery notification" ohne Platzhalter verwenden werden ausschliesslich Nachrichten mit der exakten Übereinstimmung des Betreffs "delivery notification" als Bounce-Back verarbeitet. Eine Email mit dem Betreff "delivery to xxx@yyy.com notification failure" würde dann nicht als übereinstimmend gewertet werden.

Sie können außerdem bestimmte Wörter (getrennt mit dem Komma) festlegen, welche in der Nachricht vorhanden sein müssen, um diese Nachricht als unzustellbar einzuordnen. So enthalten zum Beispiel die meisten Unzustellbarkeitsberichte das Wort "Grund" (gefolgt von einem Text, welcher den Grund des Nichtzustellens der Nachricht erläutert). Please note that the list of words is not a pattern (you cannot use wildcards) and the logical operator of the list is OR.

Falls auf eine Nachricht die von Ihnen festgelegten Regeln zutreffen, wird diese als Bounce-back behandelt und alle Email-Adressen, welche in dieser Nachricht angegeben sind werden abgemeldet, falls diese ebenfalls in der heruntergeladenen Bounce-back Liste enthalten sind. Aus diesem Grund möchten Sie sicherlich, dass alle typischen Postmasteradressen nach einem bestimmten Muster ausgeschlossen werden.

     

Beispiel

Stellen Sie sich vor, Sie möchten eine Nachricht als Bounce-back behandeln, wenn diese:

a) das Wort "undeliverable" und "message" in der Betreffzeile beinhaltet;
b) von einer "postmaster@..." Adresse kommen;
c) deren Betreffzeilentyp "delivery notification" oder "delivery failure notification" lautet
d) deren Inhalt die Worte "reason" oder "error" im Haupttext beinhaltet

und Sie natürlich den Absender ("postmaster@...") nicht abmelden möchten, da die eigentliche nicht zu erreichende Adresse im Nachrichtentext steht.

Hier sehen Sie, wie dieses Format aussieht:

Harte Rückläufer:   (überprüft)
Musterbetreff:   *undeliverable*message*
1. Header-Variable:   From
  Inhalt:   *postmaster*
2. Header-Variable:   Content-Type
  Inhalt:   *delivery*notification*
Nachrichteninhalt :   reason, error
Ausgeschlossen :   postmaster@*

Und hier noch ein einfachereres Format, welches ausgeführt werden soll, wenn das Wort "undeliverable" in deren Betreffzeile vorhanden ist (und postmaster@... und webmaster@... ausgeschlossen werden sollen:

Harte Rückläufer:   (überprüft)
Musterbetreff:   *undeliverable*
1. Header-Variable:   (empty)
  Inhalt:   (empty)
2. Header-Variable:   (empty)
  Inhalt:   (empty)
Nachrichteninhalt :   (empty)
Ausgeschlossen :   postmaster@*, majordomo@*
