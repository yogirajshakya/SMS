Historik

Här kan du se en lista på alla gjorda utskick. Om ett utskick inte avslutades (tex för att din SMTP-server vägrade ta emot mera än ett visst antal) är det möjligt att fortsätta från det ställe där avbrottet skedde, skicka till de som inte fick utskicket.

Varje utskick markeras med datum, adresslistans namn och rubriken.

Klicka på knappen Förhandsvisning (eller klicka med höger musknapp och välj) för att visa det aktuella meddelandet.

Knappen Adresslista visar listan med adresser som meddelandet skickades till. Alla adresser som meddelandet skickades till utan fel visas i grönt, och adresser i rött vissar att något fel inträffade.

Knappen Log visar den dialog som skedde med SMTP-servern (eller med mottagarens server vid direkt sändning).

Om inte utskicket gjordes korrekt till alla adresser visas symbolen längst till höger på raden. Genom att klicka på den fortsätta utskicket enligt inställningarna under Skicka. Meddelandet skickas då till alla adresser som inte fick det tidigare (de adresser som markerats med rött ).

Historikrader kan raderas genom att markera dem (använd Ctrl för att markera flera, eller Shift för att välja en grupp) och klicka på knappen Ta bort. Adresser kan exporteras genom att välja Exportera raderade eller Exportera olevererade. Att exportera olevererade adresser kan också vara användbart för att rensa adresslistorna från felaktiga adresser: Du kan senare importera filen och välja tabort eller avsluta prenumeration.

     

När skall man använda Historikfunktionerna?

Att avsluta ett avbrutet utskick är ju en bra funktion för att hantera eventuella problem med SMTP-server. Det kan ju hända att det blir problem trots Avancerade inställningar under Skicka används för att skicka försiktigt och långsamt. Tex om du har en situation där utskicket avbryts efter halva adresslistan. Sedan kan du använda Historik-funktionerna för att avsluta utskicket.

Att exportera felaktiga (röda) adresser kan också användas för att rensa adresslistan från felaktiga adresser.
