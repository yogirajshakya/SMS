Invia

Per inviare il messaggio corrente è necessario innanzitutto indicarne i destinatari ed il mittente.

Nel campo Selezionare la lista a cui inviare è presente l'elenco delle liste di distribuzione (definite in Gestione liste); è necessario sceglierne una, i cui indirizzi saranno i destinatari del messaggio. Ogni contatto riceverà una mail personalizzata.

Il mittente del messaggio è indicato nei campi Indirizzo e-mail mittente e Nome mittente. In questi campi è possibile digitare liberamente, oppure si può scegliere un indirizzo e un nome dagli elenchi a discesa che conservano gli ultimi dieci indirizzi e nomi utilizzati per precedenti invii (se non è mai stato inviato nessun mailing questi elenchi saranno vuoti). Se alla lista di distribuzione scelta è stato impostato un indirizzo e-mail per le iscrizioni, questo indirizzo verrà automaticamente proposto come indirizzo del mittente; tuttavia si tratta appunto soltanto di un suggerimento, ed è possibile digitare direttamente o scegliere dall'elenco un indirizzo e-mail differente.
E' opzionalmente possibile indicare un Indirizzo per risposte: questo sarà l'indirizzo a cui saranno inviate le risposte al messaggio inviato. Lasciando questo campo vuoto, le risposte saranno indirizzate al mittente del messaggio (questo è il comportamento standard dei messaggi personali).

Per inviare il messaggio sarà sufficiente cliccare sul tasto Invia; naturalmente se si tratta del primo invio, oppure le impostazioni della connessione sono cambiate, sarà necessario configurare preventivamente le Impostazioni di Invio, altrimenti il messaggio non potrà essere recapitato.

In alternativa all'invio immediato dei messaggi, è possibile programmare il mailing per l'invio differito, e SendBlaster lo invierà automaticamente alla data e ora stabilite; è anche possibile "parcheggiare" il mailing, senza impostare una scadenza di invio, in modo che il mailing possa essere preparato in anticipo e poi lanciato manualmente in un momento successivo. Per impostare un invio differito usare il pulsante Invia in seguito...

Vi sono due possibili Modalità di invio: Invio diretto e Usa server SMTP.

  • Selezionando Usa server SMTP l'invio delle mail avverrà esattamente come farebbe un normale client di posta elettronica, cioè consegnando la mail al server SMTP del proprio provider di connettività. E' dunque necessario specificare il nome del Server SMTP e, qualora sia necessaria l'autenticazione dell'utente, selezionare Autenticazione necessaria ed indicare Username e Password. Questi parametri sono gli stessi utilizzati nel proprio client di posta elettronica, pertanto sarà sufficiente "copiarli" da un account di posta elettronica funzionante (per assicurarsi che il server SMTP sia raggiungibile è sufficiente fare un Test connessione). La casella SSL abilita la comunicazione in modalità SSL, che può essere richiesta da alcuni server SMTP (un esempio tipico è il servizio di GMail).

    Suggerimento: Il pulsante Profili permette di gestire più configurazioni SMTP. I dati degli account SMTP possono essere salvati in più profili e poi ricaricati con pochissimi click. Questa funzionalità è utile in particolare quando si inviano mailing per conto terzi.

    Abilitando Riprova con invio diretto in caso di errore, qualora la consegna di un messaggio non sia accettata dal server SMTP il programma proverà a consegnare direttamente il messaggio al server di posta del destinatario, utilizzando le impostazioni di Invio diretto.

  • Selezionando Invio diretto il programma cercherà il server di mail di ogni singolo destinatario e si collegherà direttamente, "saltando" il server SMTP, che non sarà quindi necessario indicare. Si tenga presente che alcuni domini di posta (ad esempio hotmail) non accettano l'invio diretto: il messaggio è rifiutato se proviene da un sistema desktop anziché da un server SMTP pubblico.
    Abilitando Riprova con invio tramite SMTP in caso di errore, qualora la consegna diretta di un messaggio non sia accettata dal server di posta del destinatario il programma proverà a consegnare il messaggio al server SMTP, utilizzando le impostazioni della sezione Usa server SMTP.

     

Usare l'invio diretto o il server SMTP?

La scelta dipende innanzitutto dalla disponibilità di un buon server SMTP. Se il server SMTP fornito dal proprio provider di connettività è in grado di accettare molti messaggi in rapida sequenza, o anche l'invio di più messaggi contemporaneamente, l'utilizzo di un server SMTP è più sicuro e veloce, perché il server SMTP è più "vicino" al proprio computer rispetto ai server di mail dei singoli destinatari.

Tuttavia alcuni provider, al fine di evitare che i propri utenti usino il server per fare spamming, impongono limitazioni al numero di messaggi che possono essere inviati in sequenza entro un certo spazio di tempo ed al numero di messaggi che possono essere inviati contemporaneamente (ad esempio, un server SMTP potrebbe accettare non più di un messaggio ogni 5 secondi, e non consentire l'invio di più di due messaggi alla volta). I messaggi che eccedono questo limite saranno rifiutati con un errore. Questo non significa che il server o il programma non funzionino, ma che sarà necessario impostare in modo diverso la temporizzazione dell'invio nella sezione Impostazioni avanzate.

In questo caso però potrebbe essere più conveniente e veloce usare l'invio diretto. L'invio diretto richiede un po' di tempo in più per la consegna di ogni messaggio (dato che deve effettuare una richiesta al DNS ed il server di posta potrebbe essere molto "lontano" dal PC dell'utente), ma questo tempo addizionale è compensato dalla possibilità di inviare più messaggi contemporaneamente senza inoltre dover impostare pause fra un messaggio e l'altro.

Un secondo caso in cui l'invio diretto è consigliato è quando si debba inviare un mailing usando una connessione di cui non si conosce il server SMTP (il tipico caso di chi utilizza un notebook "in esterno", cioè non nel proprio ufficio o abitazione - si pensi al caso delle reti wireless ad accesso pubblico).

      

La sezione Impostazioni avanzate permette di configurare con precisione la temporizzazione dell'invio ed altre opzioni; questo è utile per inviare alla massima velocità possibile consentita dalle protezioni antispam del proprio server SMTP. Ecco il significato di ogni singola opzione:

Pannello Prestazioni:

  • Pausa fra i messaggi: il numero di secondi che intercorrono fra l'invio di un messaggio ed il successivo
  • Pausa fra i blocchi: il numero di secondi di attesa fra l'invio di un blocco di messaggi e l'altro
  • Messaggi per blocco: il numero di messaggi inviati in ogni blocco
  • Timeout: il numero di secondi trascorsi i quali l'invio di un messaggio viene interrotto
  • Connessioni: il numero di connessioni simultanee, cioè di messaggi inviati contemporaneamente*
  • Ulteriori tentativi in caso di errore: se il server SMTP o il server di posta del destinatario (nel caso di invio diretto) non accettano un messaggio, il programma può attendere un certo numero di secondi e ritentare l'invio una o più volte.
  • Pausa fra i tentativi: il numero di secondi di attesa prima di ritentare l'invio di un messaggio non accettato.

La giusta configurazione non può essere stabilita a priori, e spesso può essere individuata solo dopo una serie di tentativi, verificando quanti messaggi siano recapitati. Per semplificare le impostazioni sono proposte alcune Configurazioni suggerite; non è detto che siano ottimali per la propria connessione, ma possono essere utili punti di partenza per arrivare alla configurazione ottimale.

Importante: la lista di distribuzione può contenere indirizzi non più validi, oppure alcuni server di posta potrebbero essere temporaneamente inattivi. Pertanto con liste che contengano numerosi indirizzi è normale ed inevitabile che, indipendentemente dalla modalità scelta e dalle impostazioni utilizzate, non tutti i messaggi siano recapitati ed una certa percentuale di essi restituisca un errore. In questo caso sarà comunque possibile provare a completare l'invio in un momento successivo, utilizzando le funzionalità della cronologia.

     

Quali sono le giuste impostazioni avanzate?

Per trovare le giuste impostazioni avanzate è necessario fare qualche tentativo (normalmente è sufficiente una lista contenente una decina di indirizzi) e verificare quanti messaggi sono recapitati (questa informazione viene fornita al termine dell'invio, quando è anche possibile consultare i log). Gli utenti che dispongono di un server SMTP sufficientemente "tollerante" probabilmente non avranno problemi indipendentemente dalle impostazioni inserite, che avranno effetto solo sulla velocità di invio.
Al contrario, se la propria connessione dispone di un server SMTP con un filtro antispam piuttosto "severo", per riuscire a consegnare un numero elevato di messaggi potrebbe essere necessario rallentare di molto l'invio e non inviare più messaggi contemporaneamente.

Ecco alcune considerazioni generali:

  • Un server SMTP tollerante consente un invio rapido e senza pause (o comunque con pause limitate); su un server SMTP che utilizzi numerosi filtri antispam, in caso di problemi è necessario aumentare le pause e diminuire il numero di connessioni.
  • I server SMTP di norma non consentono molti invii simultanei, pertanto usando la modalità SMTP il numero di connessioni deve essere basso o, nei casi più critici, pari a 1 (nessuna connessione simultanea). Se il server SMTP utilizza meccanismi di protezione antispam, sarà necessario rallentare l'invio impostando una pausa di qualche secondo fra un messaggio e l'altro, ed impostare una pausa più lunga fra i blocchi di messaggi; i blocchi inoltre non dovranno contenere molti messaggi (qualche decina).
  • Nel caso dell'invio diretto, in una lista tipica, che contenga mail appartenenti a differenti domini, i server di destinazione sono comunque diversi tra loro: è quindi possibile impostare un numero di connessioni elevato* e pause limitate (o nessuna pausa) fra un messaggio e l'altro e fra i diversi blocchi.
  • Aumentare il numero di tentativi in caso di errore può aumentare la percentuale di successo, ma rallenterà l'invio, soprattutto se è basso il numero di connessioni simultanee. Una strategia alternativa è quella di completare l'invio in un momento successivo (lasciando trascorrere qualche decina di minuti o qualche ora) dalla scheda cronologia.
      

Pannello Extra:

  • Conferma di lettura: imposta la richiesta della conferma di lettura (il destinatario è comunque libero di inviarla o non inviarla)
  • Priorità massima: notifica al destinatario che il messaggio è importante
  • Testo XMailer: indica il programma che ha inviato il messaggio (Testo Xmailer): usando l'impostazione Default l'header riporterà il nome e la release di SendBlaster; in alternativa è possibile digitare un testo libero, per simulare l'invio con un altro programma di posta elettronica, oppure scegliere l'opzione Nessuno per evitare che l'header venga inserito nel corpo della mail
  • Dominio: consente di personalizzare il nome con cui il computer si connette al server SMTP o al server di posta del destinatario. Questa opzione può essere utile quando si usa l'Invio diretto: alcuni server di posta, infatti, al fine di limitare lo spam effettuano un controllo sul dominio; in particolare richiedono che il dominio corrisponda all'hostname con cui il computer è visibile in rete, e prima di accettare il messaggio eseguono un ping (o un'operazione equivalente) per verificare che il nome del dominio corrisponda all'indirizzo IP del computer che richiede la connessione. In questo caso selezionare l'opzione Indirizzo internet PC può far sì che un numero maggiore di messaggi venga accettato. Si consiglia di conservare le impostazioni predefinite, oppure utilizzare l'opzione Indirizzo internet PC per l'invio diretto e l'opzione Nome locale PC per l'invio tramite SMTP (i server SMTP di norma non controllano il dominio). L'indirizzo pubblico viene controllato da SendBlaster ogni volta che si avvia il programma; è tuttavia possibile che, qualora al momento dell'avvio la connessione non fosse disponibile, l'indirizzo pubblico resti vuoto (in questo caso sarà usato automaticamente il nome locale). Se il computer dispone di un indirizzo IP pubblico fisso, gli utenti esperti potranno quindi usare l'opzione Specifica... ed inserire un valore fisso (ricordando che è necessario indicare non l'IP pubblico ma il corrispondente hostname) - un'opzione consigliata solo a chi sa esattamente cosa sta facendo perché un errore potrebbe comportare l'impossibilità di recapitare i messaggi
  • Specifica manuamente server DNS: SendBlaster per default utilizzerà il DNS predefinito della connessione attualmente utilizzata, ma è anche possibile (ed in alcuni casi può essere necessario) indicare un DNS differente, abilitando questa opzione ed inserendovi il relativo indirizzo IP
  • Ignora errori socket: normalmente, quando si verifica un problema con la connessione ad Internet, a causa di errori hardware o software, SendBlaster interrompe il mailing ed avverte l'utente, restando in attesa di istruzioni - nella maggior parte dei casi infatti è richiesto un intervento manuale per ripristinare la connessione (ad esempio resettando le apparecchiature di rete, etc). Abilitare questa opzione se invece non si desidera che il mailing venga interrotto, e si preferisce che SendBlaster ignori gli errori (tenendo ben presente che, se la connessione ad Internet non funziona, il mailing non andrà comunque a buon fine). Questo può essere utile in caso di una connessione instabile che tende a cadere periodicamente ma viene ripristinata automaticamente in breve tempo (ad esempio, in caso di interferenze in una connessione wi-fi).

Pannello Intestazioni aggiuntive:

Talvolta gli utenti avanzati potrebbero voler aggiungere intestazioni (header) speciali al messaggio, che consentano di ottenere caratteristiche speciali (monitoraggio, firma, etc). SendBlaster consente di aggiungere fino a un massimo di dieci intestazioni personalizzate al messaggio.

*Nell'edizione Free il numero massimo di connessioni contemporanee è 2; anche impostando un numero superiore verranno comunque utilizzate al massimo due connessioni.
